# ORDL R&D Program Map

## Active Tracks

1. Platform Reliability
- Objective: keep inference and storage paths stable under load.
- Outputs: smoke tests, regression tests, incident postmortems, runtime hardening.

2. Text Intelligence
- Objective: improve summarization quality, domain extraction, and labeling reliability.
- Outputs: benchmark set, prompt/task profiles, tuned model candidates.

3. Vision Intelligence
- Objective: improve class fidelity and detection recall for ORDL-relevant imagery.
- Outputs: curated image corpus, error taxonomy, fine-tuning experiments.

4. Anomaly Intelligence
- Objective: reduce false positives and improve temporal explainability.
- Outputs: labeled time-series sets, threshold policy, performance curves.

5. Knowledge and Retrieval
- Objective: maximize traceability and retrieval usefulness for decisions.
- Outputs: richer schemas, ingestion coverage reports, retrieval evaluations.

6. Publication and Reproducibility
- Objective: publish meaningful technical outputs from each release cycle.
- Outputs: whitepaper updates, benchmark snapshots, model cards, release notes.

## Milestones

1. M1 - Evidence-complete ingestion
- All internal ORDL docs indexed with ingestion manifests.
- Current AI research ingested on recurring cadence.

2. M2 - Evaluation baseline
- Fixed eval suite for text, image, and anomaly tracks.
- Baseline metrics versioned in repo.

3. M3 - ORDL-tuned experts
- Initial fine-tuned model candidates with model cards and eval deltas.

4. M4 - Publication cadence
- Monthly research brief and tagged release report.
