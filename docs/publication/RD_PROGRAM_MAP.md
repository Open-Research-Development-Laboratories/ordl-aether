# ORDL R&D Program Map

Back to index: [../INDEX.md](../INDEX.md) | [Publication Program](README.md)

## Program Objective

Build AETHER into a durable research platform that produces measurable technical progress, defensible model decisions, and publication-grade artifacts.

## Active Tracks

1. Platform reliability
- Maintain stable inference and storage behavior under realistic load.
- Outputs: regression suites, smoke test coverage, incident analyses, runtime hardening.

2. Text intelligence
- Improve summary quality, extraction precision, and label calibration.
- Outputs: benchmark corpus, task profiles, model candidate comparisons.

3. Vision intelligence
- Improve class fidelity and detection recall for target imagery domains.
- Outputs: curated image sets, error taxonomy, tuning experiments.

4. Anomaly intelligence
- Improve false positive control and temporal explainability.
- Outputs: labeled time-series sets, threshold policies, detection performance curves.

5. Knowledge and retrieval
- Improve traceability and retrieval relevance across stored analyses.
- Outputs: schema expansions, ingestion coverage reports, retrieval quality evaluations.

6. Publication and reproducibility
- Convert engineering progress into externally credible technical communication.
- Outputs: white paper updates, benchmark snapshots, model cards, release briefs.

## Milestones

1. M1: Evidence-complete ingestion
- Internal ORDL documents indexed with reproducible manifests.
- External AI research corpus captured on a repeatable cadence.

2. M2: Evaluation baseline
- Fixed evaluation suites for text, vision, and anomaly tracks.
- Baseline metrics versioned in repository.

3. M3: ORDL-tuned experts
- Initial domain-tuned model candidates with full model cards.
- Before and after metric deltas documented by release.

4. M4: Publication cadence
- Recurring research briefs with linked benchmarks and release notes.
