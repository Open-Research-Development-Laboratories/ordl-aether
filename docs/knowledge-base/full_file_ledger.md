# File Ledger Policy

This repository uses an internal file-ledger process for integrity and reproducibility checks.

## Public Scope

- Public documentation includes policy, architecture, and release-safe summaries.
- Full per-file hash ledgers and internal inventory exports are retained in private ORDL archives.

## Rationale

- Prevents accidental disclosure of internal structure and operational tooling.
- Keeps publication materials focused on research outcomes and technical quality.
- Preserves internal auditability without exposing private implementation details.
